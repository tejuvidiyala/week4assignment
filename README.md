# Azure Cloud Fundamentals and Azure Data Factory Pipeline Implementation

## Overview

As part of my Data Engineering Internship at Celebal Technologies, I worked on understanding the basic concepts of Microsoft Azure and Azure Data Factory by building a simple end-to-end data pipeline. The main objective of this project was to learn how data can be stored, validated, and moved between Azure services using Azure Data Factory.

During this assignment, I created Azure resources, configured storage services, built datasets and pipelines in Azure Data Factory, executed the pipeline, verified the output, and explored role-based access control using IAM.

## What I Implemented

### Resource Group Creation

I started by creating a Resource Group in Azure, which acted as the central container for managing all the resources used in this assignment.

### Storage Account and Blob Containers

After creating the Resource Group, I created an Azure Storage Account and then created two Blob containers: source-container and destination-container.

The source container was used to store the input dataset, while the destination container was used to store the output generated by the pipeline.

I uploaded the Sample Superstore CSV dataset into the source container for further processing.

### Azure Data Factory Setup

I created an Azure Data Factory instance and explored its Author, Monitor, and Manage sections to understand how data integration pipelines are designed and monitored.

I then created a Linked Service to establish a connection between Azure Data Factory and Azure Blob Storage.

After creating the Linked Service, I created source and destination datasets that were used by the pipeline activities to access the input and output files.

### Metadata Validation

To understand how file validation works before processing data, I used the Get Metadata activity. This activity was configured to check properties such as file existence, structure, and other metadata information.

This helped me understand how metadata validation can be used in real-world projects to avoid processing invalid or missing files.

### Pipeline Development

I created a pipeline named SuperstorePipeline and added the required activities to move data from the source container to the destination container.

The pipeline first validates the source file and then performs the data copy operation. The activities were connected using success dependencies to ensure that each step executes only after the previous step completes successfully.

### Pipeline Execution and Verification

After configuring the pipeline, I executed it using the Debug option in Azure Data Factory Studio.

The pipeline ran successfully, and I verified that the output file was generated in the destination container. I also monitored the execution status and reviewed the activity outputs to confirm successful execution.

### IAM Role Assignments

I explored Azure IAM (Identity and Access Management) and assigned the required roles for accessing resources.

The Reader role was assigned successfully, and the Storage Blob Data Contributor role was assigned to the Azure Data Factory managed identity to enable read and write access to Blob Storage.

While attempting the Contributor role assignment, the role was not visible in the Azure for Students environment, which appears to be a subscription-level limitation.

## Mini Project: Metadata-Validated Pipeline

As an extension of the assignment, I created a metadata-validated pipeline that performs a validation check before copying data.

The pipeline follows the workflow:

Get Metadata → If Condition → Copy Data

The Get Metadata activity validates the source file, and the If Condition activity ensures that the file satisfies the required conditions before allowing the Copy Data activity to execute.

After execution, all activities completed successfully, and the output file was created in the destination container.

This implementation helped me understand how metadata-driven validations can improve the reliability of data pipelines by preventing invalid data from being processed.

## Insights Gained

Through this assignment, I got a better understanding of how different Azure services work together to build a data pipeline. Before this project, I only had theoretical knowledge of Azure, but after performing the tasks myself, I understood how resources are created, connected, and managed in a cloud environment.

One of the most useful things I learned was the importance of validating data before processing it. By using the Get Metadata activity, I understood how Azure Data Factory can check whether a file exists and verify its properties before moving data. This helped me understand why validation is an important step in real-world data pipelines.

I also learned how Linked Services and Datasets work in Azure Data Factory and how they are used to connect storage resources with pipeline activities. Creating and configuring these components gave me practical experience in designing data workflows.

Another important learning was understanding IAM roles and permissions. While working on role assignments, I learned how access to Azure resources is controlled and why proper permissions are required for services such as Azure Data Factory to access storage accounts.

Overall, this assignment helped me gain hands-on experience with Azure services and gave me a clear understanding of the basic concepts involved in data engineering workflows.


## Challenges and Limitations

While working on this assignment, I faced a few challenges that helped me understand Azure services more deeply. One of the main challenges was configuring datasets and pipeline activities correctly in Azure Data Factory, as even small configuration issues could lead to validation errors during execution.

During the IAM role assignment task, I was able to assign the Reader role and the Storage Blob Data Contributor role successfully. However, the Contributor role was not visible in my Azure for Students subscription environment, which appeared to be a permission or subscription-level limitation.

Another limitation of this project is that it was performed using a sample CSV dataset with a relatively small volume of data. In real-world scenarios, data pipelines typically process much larger datasets and may involve additional components such as data transformation, scheduling, monitoring, logging, and error handling.



## Conclusion

This project provided me with hands-on experience in building and managing data pipelines using Azure Data Factory. I learned how different Azure services work together to store, validate, and move data efficiently. The assignment also helped me understand metadata validation, access management, and pipeline orchestration, which are important concepts in Data Engineering.
